# Project-Scoped Rules for ZiadDorrah/Track

## SDLC Git Branching Instructions
- After implementing any code changes, bug fixes, or new features, the agent MUST provide the user with clear instructions and the exact Git commands required to manage these changes across the project's SDLC branches.
- This includes:
  - Instructions on creating and working in a `feature/` or `hotfix/` branch.
  - Instructions on merging changes into `develop` or `staging` for testing.
  - Instructions on releasing changes to `master` and tagging versions.
